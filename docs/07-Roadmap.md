# 07 · Roadmap

Cada fase es **usable por sí sola**; no hay que esperar al final para tener algo que funcione.

## Fase 0 — Diseño *(actual)*
Esta documentación. Decidir el hospedaje del KB y el set mínimo de conectores. → ver [09](09-Decisiones-Abiertas.md).

## Fase 1 — MVP portátil
Skill renombrado (`glossa`) + 2–3 conectores (un buscador tipo Exa/Tavily + OpenAlex + GitHub) + web search. Flujo completo de extremo a extremo: semilla → investigación → compuerta → MDX bilingüe → *push*. Probarlo con **una pieza real** (p. ej. PATTERN). Aún sin KB.

## Fase 2 — KB
Indexar la biblioteca, conector de recuperación, "primero-KB" en el router, y el **volante** (las piezas vuelven al corpus).

## Fase 3 — Fuentes legales
CourtListener, GovInfo, *Federal Register*, etc., para el track de mayor interés.

## Fase 4 — Vigilancia
Tareas programadas → digest automático → "¿lo convertimos?".

## Fase 5 — Vitrina+
Páginas de procedencia/método en el sitio: enseñar la semilla, las fuentes y el rastro de cada pieza. Migración de marca a Glossa completada (repo, sitio, dominio).

---

**Primer paso concreto:** cerrar una decisión de la Fase 0 (hospedaje del KB o qué buscador entra primero) y arrancar la Fase 1 con la pieza de prueba.
